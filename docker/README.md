# Docker

## Image

The Dockerfile builds a multi-arch node image based on Ubuntu 24.04 with:

- AnvoIO Core v0.1.0-alpha (core_netd, core-cli, core-wallet, core-util)
- jq, zstd, btrfs-progs, socat, gosu, cron, rclone

Supports both `amd64` (x86_64) and `arm64` (Apple Silicon / AArch64) via Docker buildx.

Build argument `CORE_VERSION` controls the AnvoIO Core version (default: `0.1.0-alpha`).

```bash
# Native architecture
docker build -t core-node:0.1.0-alpha -f docker/Dockerfile docker/

# Cross-platform (amd64 + arm64)
docker buildx build --platform linux/amd64,linux/arm64 -t core-node:0.1.0-alpha -f docker/Dockerfile docker/
```

The start script (`scripts/node/start.sh`) builds the image automatically if it doesn't exist.

## Entrypoint

`entrypoint.sh` runs as the container entrypoint:

1. Ensures required directories exist (`/opt/core/{config,data,logs,data/snapshots}`)
2. Auto-detects the latest snapshot in the snapshots directory
3. Drops privileges to the `core` user via gosu
4. Executes the core_netd command passed as arguments

## Docker Compose

The compose file is generated by `scripts/setup/generate-config.sh` from the template at `config/templates/docker-compose.yml.tmpl`. It is placed at `$STORAGE_PATH/config/docker-compose.yml`.

Key properties:
- `network_mode: host` — direct port binding, no Docker networking overhead
- `stop_grace_period: 30m` — allows core_netd to flush state cleanly
- tmpfs mount for `/opt/core/data/state` when `STATE_IN_MEMORY=true` (sized from `CHAIN_STATE_DB_SIZE + 10%`)
- Health check via `/v1/chain/get_info` (non-seed roles)
- Restart policy from `node.conf`

## No Manual Editing

The compose file and all configs are generated from `node.conf`. Do not edit them directly — changes will be overwritten on the next `generate-config.sh` run. Edit `node.conf` instead.
